# Skill: hackathon-autopilot

## Purpose

Generate finalist-grade hackathon delivery:

- novel, wow ideas aligned to event criteria
- time-consistent MVP scope
- architecture + build plan
- VC-ready pitch + applause-first demo
- risk cuts & fallbacks
- auto UI flow spec for Figma
- multi-agent orchestration plan
- winner patterns analysis to bias toward what wins

## Always-On Modes

✓ Trend-Aware Mode
✓ VC Mode
✓ Demo Domination Mode
✓ Jury Psychology Mode
✓ Winner Patterns Mode
✓ Auto Figma UI Flow Mode
✓ Multi-Agent Orchestration Mode

## 5 Mandatory Impact Criteria

1. human behavior insight
2. real-time decision support
3. invisible → visible
4. emotional connection
5. instant demo clarity

## Guardrails

- Do NOT invent event facts.
- Ask user to paste missing timeline/criteria.
- Prefer demo-first feasibility and offline-safe fallback.

## Execution Pipeline

1. Event Brief Extraction
2. Winner Patterns analysis
3. Generate 6 ideas → filter generic
4. Score with 5-criteria gate + winner-pattern bonus
5. Shortlist Top 3
6. Pick Winner
7. Winner Pack (architecture, build plan, pitch, demo, risk, alignment, win DNA)
8. Auto Figma UI Flow (screens, components, prompt)
9. Multi-Agent Orchestration Plan (roles, handoffs, checkpoints)
